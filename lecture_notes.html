Intro to Torch

Torch is currently one of the most popular deep learning frameworks in top research labs around the world, such as Facebook and Google especially DeepMind. Torch started from Y. LeCun's lab, who is one of the leading experts in deep learning and the reference when working with convolutional neural networks. In Torch there is a great array of existing codes, deep learning softwares and components that will make your life easy, especially once you successfully complete this course. Torch sits on top of an existing programming language, Lua. Lua is a lightweight, very fast multi-paradigm programming with a somewhat "Matlab"-like coding style. Torch and Lua are rich in features and relatively easy to use and, therefore, are ideal for rapid prototyping. However, they can be quite annoying in the beginning, because of some peculiarities. Although Torch does not come with a graphical interface that would allow for easier programming, it does have a web interactive interface, namely iTorch. For those of you familiar with Jupyter notebooks for iPython, this should be quite easy. Jupyter has implemented a Torch kernel, therefore, one can easily run torch software interactively on his/her web-browser as if he was running Matlab. For those of you not familiar with Jupyter notebooks,

The Jupyter Notebook is a web application that allows you to create and share documents that contain live code, equations, visualizations and explanatory text. Uses include: data cleaning and transformation, numerical simulation, statistical modeling, machine learning and much more.
The Jupyter Team
The great thing about Jupyter notebooks is that you can start them as easily as just typing itorch notebook in your terminal (of course once you have iPython and iTorch installed). Next, we will guide you through the installation process and provide you with some short tutorial in Lua, and a longer tutorial in Torch.

Starting with Lua

A useful Lua tutorial can be found here: http://tylerneylon.com/a/learn-lua/

After you read these tutorials, you should be able to start programming with Lua. Before continuing, you should be confident that you can answer the following questions:

  • When do we need to use the keyword local? What is the default scope in Lua?
  • What is the difference between a.f() and a:f()?
  • What does require do? Why do we sometimes need to capture its value in a variable?
  • What is the Lua equivalent of a list object? How do we access or iterate over elements in such a construction?

Note that local variables only exist in standalone scripts that one runs from the terminal. If you just start an interactive session with th, local variables do not make sense. If you define one, you will simply not be able to access it.

Starting with Torch

Torch generalizes in an efficient manner vectors (1-d) arrays and matrices (2-d) arrays to tensors (\texttt{torch.Tensor()}) of $n$-dimensions. Some links with documentation on the most important Torch classes of objects are:

In large scale machine learning research memory and computational efficiency can be of paramount importance. With Torch one can achieve high efficiency if one performs as many operations as possible in-place. This means, you should try to avoid to define/call new variables, if you can reuse existing ones. Trhough specialized functions Torch allows this to be done quite easily. For example consider the following case.

In the first example a new variablel, \texttt{t3}, is defined and memory allocation is performed. If, however, \texttt{t}, is not needed anymore, we can just overwrite it to store the final summation. As a result, we avoid allocating new resources. Although this example is trivial, you might want to rethink it when you have matrices with millions of entries. In general there are specialized functions for avoiding to performing specific tensor operations manually. For example \texttt{torch.expand([...])} expands a tensor without allocating memory and \texttt{torch.fill([...])} fills in matrix entries without needing to run a for loop.
